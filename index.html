容器控制器能够将多个控制器中的内容合并到一个单独的用户界面。容器控制器常被用于方便导航以及基于存在的内容创建新的用户界面。UIkit中的容器控制器包括 UINavigationController, UITabBarController,UISplitViewController。这些都方便用户不同界面的引导。

设计一个容器控制器
多数情况下，容器控制器类似于其他内容控制器，它管理一个跟视图和一些内容。
不同的是，容器控制中的从其他控制器中获得内容。它获得的内容只限于其他控制器的view，这些控制器在它的view层级中。内容控制器设置包含views的尺寸和位置，但是初始的控制器依旧管理这些内容。

当你设计自己的容器控制器的时候，理解容器控制器和被包含的控制器的区别。这些控制器的关系可以帮助如何在屏幕上显示他们的内容，以及容器控制器内部管理它们。在设计过程中，回答下列问题。

1.容器控制器和它包含的控制器各自的作用
2.同时展现多少个子控制器。
3.包含的控制器之间的关系。
4.控制器如何添加和移除。
5.子控制器的size和position能够改变么。什么情况下会改变。
6.容器控制器提供辅助或者引导类似的view么
7.容器和子控制器需要的通信方式？容器控制器需要报道特定的事件，不同于UIViewController定义的标准。
8。容器控制器能以不同的形式展示么？如果能，怎么设置？

在你定义了多个对象后，容器控制器的实现就相对直观了。UIKit唯一的要求是你在容器控制器和子控制器建立了正式的parent-child关系。parent-child关系保证children可以收到所有系统消息。除了这个，大多数的工作发生在容器控制器的布局和管理中，每个控制器都不一样。你可以在你的容器控制器中将view放置在你想要的任何位置。你可以添加自定义的view来提供辅助或者引导功能。

实现一个容器控制器。
为了实现一个容器控制器。你必须建立容器控制器和子控制器的关系。在你试图管理这个子控制器前必须建立这些parent-child关系。这么做让UIKit知道你的控制器正在管理子控制器的size和position。你可以在Interface Builder或者用代码创建这些关系。当使用代码创建parent-child关系时，你需要明确的添加或者删除子控制器作为控制器设置的一部分。

添加一个子控制器
为了用代码包含一个子控制器。创建相关控制器的parent-child关系，如下
1.执行容器控制器的addChildViewController:方法
2.添加子控制器的根view到容器控制的view层级，记住设置子控制器的size和position作为这个过程的部分。
3.可以添加约束，用来管理子控制器view的size和postion
4.执行子控制器的didMoveToParentViewController方法

Listing 5-1 展示了一个容器控制器如何包含一个子控制器的。在建立了parent-child关系之后，容器控制器会设置子控制器viewdeframe，并且将它添加到自己的view层级中。设置子类的frame是重要的，并且保证view正确的显示在容器中。在添加view之后，容器调用子类的didMoveToParentViewController: 方法来给子类控制器一个机会来相应view所有权的变化。

  • (void) displayContentController: (UIViewController*) content {
    [self addChildViewController:content];
    content.view.frame = [self frameForContentController];
    [self.view addSubview:self.currentClientView];
    [content didMoveToParentViewController:self];
    }

在上面的例子中,你只是执行了子控制器的didMoveToParentViewController，这是因为addChildViewController方法会调用子控制器的willMoveToParentViewController:。你必须调用didMoveToParentViewController的原因是这个方法不会执行直到你将子控制器的view包含在你的view层级。

如果使用AutoLayout,在将child添加到容器之后设置约束，并且你的约束只影响子控制器的根view的size和position
不要影响子控制器根view的子类

移除一个子控制器
为了移除子控制器，移除控制器间的parent-child关系，做如下操作。
1.执行子控制器的willMoveToParentViewController: 方法，参数为nil.
2.移除子控制器根view的约束
3.移除子控制器的view
4.执行子控制器的removeFromParentViewController来结束parent-child关系

移除子控制器永久的影响父控制器和子控制器的关系。当你不再需要子控制器时，移除子控制器。比如，一个导航控制器不会移除现在的控制器，当它跳转到一个新的页面时。只有当它们移出栈的时候才会被移除。

List5-2展示了如何从container移除子控制器。调用子类的willMoveToParentViewController：，参数nil可以给子控制器一个机会准备这个改变。removeFromParentViewController也会调用子类的didMoveToParentViewController:方法，参数为nil，设置父控制器为nil最后确定了子控制器的移除。

  • (void) hideContentController: (UIViewController*) content {
    [content willMoveToParentViewController:nil];
    [content.view removeFromSuperview];
    [content removeFromParentViewController];
    }

子控制器间的过渡。
当你希望通过动画来替换另一个控制器。将移除和添加子控制器纳入过渡动画进程。在动画之前，确保子控制器都是你的内容，但是让当前的控制器直到，它将移走了。在动画过程中，移动新子控制器的view并且移除旧控制器的view，在动画结束时，完成子控制器的移除。

List 5-3 展示了如何使用过度动画来从一个控制器过度到另一个控制器。在这个例子中，新的控制器通过动画占据现在存在的控制器，这个控制器会被移除屏幕。在动画结束后，completion block移除了容器中的额子控制器。在例子中，transitionFromViewController:toViewController:duration:options:animations:completion: 方法自动更新了容器的view层级，所以不需要再移除或者添加views

  • (void)cycleFromViewController: (UIViewController) oldVC
    toViewController: (UIViewController    ) newVC {
    // Prepare the two view controllers for the change.
    [oldVC willMoveToParentViewController:nil];
    [self addChildViewController:newVC];
    // Get the start frame of the new view controller and the end frame
    // for the old view controller. Both rectangles are offscreen.
    newVC.view.frame = [self newViewStartFrame];
    CGRect endFrame = [self oldViewEndFrame];
    // Queue up the transition animation.
    [self transitionFromViewController: oldVC toViewController: newVC
    duration: 0.25 options:0
    animations:^{
    // Animate the views to their final positions.
    newVC.view.frame = oldVC.view.frame;
    oldVC.view.frame = endFrame;
    }
    completion:^(BOOL finished) {
    // Remove the old view controller and send the final
    // notification to the new view controller.
    [oldVC removeFromParentViewController];
    [newVC didMoveToParentViewController:self];
    }];
    }

管理子控制器的界面更新

当容器添加了一个子控制器，容器控制器自动发送界面相关的信息给子控制器。这是你希望的行为，因为这确保了所有事件合理的发送。但是，有时候，默认的行为有序的发送这个事件，虽然这对容器控制器不是很有意义。比如，如果多个子控制器同时改变它们的状态，你想加固这些改变。目的是使界面的改变可以以一种更合理的发生。

为了接管界面回调的责任。重写容器的shouldAutomaticallyForwardAppearanceMethods 方法，返回NO，
返回NO使得UIKit通知子控制器在界面的变化

  • (BOOL) shouldAutomaticallyForwardAppearanceMethods {
    return NO;
    }

当界面过度开始的时候，合理的执行子类的 beginAppearanceTransition:animated: or endAppearanceTransition。如果你的容器有一个child属性。你的容器将这些信息发送给子控制器

-(void) viewWillAppear:(BOOL)animated {
[self.child beginAppearanceTransition: YES animated: animated];
}

-(void) viewDidAppear:(BOOL)animated {
[self.child endAppearanceTransition];
}

-(void) viewWillDisappear:(BOOL)animated {
[self.child beginAppearanceTransition: NO animated: animated];
}

-(void) viewDidDisappear:(BOOL)animated {
[self.child endAppearanceTransition];
}

建立一个容器控制器的建议

  1. 只访问子控制器的根view。访问子控制器的view，不访问子控制器的其他view
